After VPC and other setup module is created using TERRAFORM-AWS-VPC and called here via TERRAFORM-AWS-VPC module.

Now create one security group , eg: roboshop-dev with only outbound access under our VPC 
and create a EC2 instance, eg: roboshop-dev and map it to roboshop-dev-private-us-east-1a subnet, 
we get only private ip as its launched in private subnet.

Now create one more SG , eg: default with only outbound access and create one ec2 instance, eg: default and map to any subnet, here also 
we get only private ip if its launched in private subnet.

Now check Peering connections in VPC and requester VPC->roboshop-dev(10.0.0.0/16) is connected to acceptor VPC ->default(172.31.0.0/16).

and also in Route tables, acceptor VPC named default (172.31.0.0/16) has connection with requester VPC named roboshop-dev (10.0.0.0/16) 
and vice-versa, i.e. requester VPC named roboshop-dev (10.0.0.0/16) related named route tables such as roboshop-dev-database, 
roboshop-dev-private and roboshop-dev-public has connection with acceptor VPC named default (172.31.0.0/16).

We require Bastion/Jump Host in public subnet to connect to ec2 instances in private subnets for security purpose.
Bastion/Jump Host - An ec2 instance in public subnet to connect to ec2 instances in private subnets for security purpose.

Now create a ec2 instance with name bastion and also create security group with inbound as My IP address, i.e. 
here IP address is your modem IP addresss and give it, this is used so that only you can login to bastion ec2 instance.

Now go to roboshop-dev ec2 instance and edit SG and in inbound rules, add a rule by giving type as SSH, port 22 and give bastion IP private address
but if bastion ec2 instance is deleted, then again its private IP address will change, then it is difficult to change IP address in all SG, 
hence give bastion SG name directly in source instead of bastion IP private address.

Now in bastion ec2 instance, check telnet eg: <roboshop-dev-ec2-instance-privateip> <portno>, eg:10.0.21.155 22 to verify if roboshop-dev ec2 
instance will accept traffic from port 22 which has instance, i.e. here Bastion ec2 instance attached to roboshop-dev-bastion SG.
